# llms.txt as an AI-readable entry point for personal websites

日期：2026-05-11  

分類：AI Signal Note / AI-readable Content  

狀態：Drafted as GitHub note

---

## 1. Context

- AVIN 已完成 Personal Website v1
- GitHub Pages 首頁已公開
- `/llms.txt` 已公開可讀
- LinkedIn 已接上網站入口
- 這則 note 來自第一週 AI Signal Review

---

## 2. Signal Summary

`llms.txt` 可以作為一份輕量的 AI 導讀檔，幫助 LLM 理解網站定位、專案結構、關鍵連結與公開數位足跡。

---

## 3. Why It Matters

- 個人網站不再只給人類瀏覽
- AI / LLM 也可能成為讀者
- 公開作品集需要同時具備 human-readable 與 AI-readable 結構

---

## 4. Workflow Implication

- 發布個人網站時，不只建立首頁
- 同步建立 `llms.txt`
- 將 GitHub repo、LinkedIn、網站、project logs 串起來
- 讓 AI 更容易理解這個人與專案的上下文

---

## 5. AVIN OS Layer Mapping

- GitHub = evidence layer
- LinkedIn = exposure layer
- Website = entry layer
- `llms.txt` = AI-readable guide layer
- Markdown = source of truth
- HTML = public display layer

---

## 6. Test Idea

- 讓 AI 分別讀首頁與 `llms.txt`
- 比較 AI 是否能更準確理解 AVIN 的定位與專案結構
- 未來可作為 AEO / AI-readable content 的小型實驗
