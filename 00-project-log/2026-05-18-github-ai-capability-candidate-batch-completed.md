- 看到熱門 repo 就想看

- 還沒分流就想裝

建立 candidate batch 的目的，是讓這些 GitHub 候選先進入可治理的候選池，而不是直接進入操作層。

## 4. 與 AI Trend Intake Layer / open-source-vault / Practical Trial Lane 的關係

這份 candidate batch 的位置在前段：

- `AI Trend Intake Layer` 幫忙判斷 signal 與 architecture fit
- `open-source-vault` 提供治理容器

因此 candidate batch 不是導入清單，而是分流清單。
