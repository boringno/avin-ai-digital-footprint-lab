# Personal Website Homepage Optimization Checklist

日期：2026-05-11  

分類：平台輸出 / Website Optimization  

狀態：首頁優化規劃清單

---

## 1. Hero 區塊

- `H1` 目前清楚，能直接傳達「公開建構中的 AI Workflow Explorer」。
- 第一眼已能大致理解 AVIN 的方向，但還可以更快回答「AVIN 是誰」與「他正在做什麼」。
- `AI Workflow Explorer / AI Operations Strategist` 的組合目前合理，但在首頁首屏中兩個稱呼同時出現，記憶點略分散。
- 目前副標偏完整敘述，資訊足夠，但節奏稍長。
- 建議評估是否補一條更有記憶點的 tagline，例如更聚焦「把 AI 學習與工具實測轉成公開數位足跡」。

結論：
- 方向正確
- 不需要大改
- 適合下一輪做小幅 wording 微調，提升第一眼辨識度

---

## 2. About / Bio 區塊

- 目前已清楚說明 AVIN 是 AI Workflow Explorer / AI Operations Strategist。
- 已成功把過去的內容、行銷、短影音、拍攝剪輯、社群經營、Meta 投放、數據優化背景，轉譯成 AI 工作流與公開輸出的底層優勢。
- 目前沒有過度醫美化，也沒有寫成 AI 大師。
- 目前也沒有落入單純 AI 工具搬運者的描述。
- 段落內容完整，但略長；若以首頁閱讀效率來看，可以再切成更短、節奏更明確的版本。

結論：
- 定位正確
- 背景轉譯合理
- 下一輪可優化句長與段落節奏，讓手機閱讀更順

---

## 3. CTA / 連結

- GitHub repo 入口明確，首頁主 CTA 清楚。
- LinkedIn 入口明確，目前在入口卡與 footer 都可看到。
- `llms.txt` 在內文中被提及，但目前沒有明確、可點擊的專屬入口。
- 目前沒有 `Contact` / `Collaboration` CTA；如果未來希望承接合作或對話，可補一個低摩擦版本。

建議檢查點：

- 是否新增 `llms.txt` 卡片或次要 CTA
- 是否新增 `Contact` / `Work with me` / `Collaboration` 類型入口
- 是否在 footer 補上更直接的網站導覽連結

結論：
- GitHub / LinkedIn 已足夠
- `llms.txt` 與 Contact 類 CTA 是下一輪最值得補的缺口

---

## 4. Projects / Experiments

- 目前首頁已有 `Current Focus`，但還沒有真正的 `Projects` 或 `Experiments` 區塊。
- 如果下一輪希望網站更像公開實驗室，而不只是定位首頁，新增 `Projects` 區塊會很有幫助。
- `AI Digital Footprint OS` 的核心模組值得逐步視覺化，例如：
  - GitHub Public Lab
  - AI Digital Footprint OS
  - Stage Logs
  - Workflow Experiments
  - llms.txt / AI-readable layer
- 也可以考慮連到：
  - Stage logs
  - docs-index
  - GitHub repo

結論：
- 目前首頁還偏入口頁
- 下一輪若要提高說服力，Projects / Experiments 區塊應列為優先候選

---

## 5. SEO / AEO / AI-readable

- `title` 目前清楚，能反映 AVIN 與核心定位。
- `meta description` 目前方向正確，但未必是最精煉版本，之後可再壓縮語句節奏。
- JSON-LD 目前仍合理，沒有誇大成 Founder、CEO 或 AI Master。
- `llms.txt` 與首頁目前方向一致，都在描述 AI Digital Footprint OS、公開建構與可驗證輸出。
- 未來可以評估新增：
  - `sitemap.xml`
  - `robots.txt`
  但目前只列為 future enhancement，不需要現在實作。

結論：
- SEO / AEO 基礎已成立
- 下一輪重點不是補新技術，而是讓首頁文案與 `llms.txt` 之間更緊密對齊

---

## 6. Mobile Readability

- 目前頁面結構已具備基本 mobile-friendly CSS。
- 主要風險不是版面壞掉，而是文字密度偏高。
- Hero 與 About / Bio 的段落在手機上可能仍稍長。
- CTA 按鈕本身清楚，但第一屏資訊量仍略多。

建議檢查點：

- 是否把長句拆成更短段落
- 是否讓 Hero intro 再縮短一點
- 是否讓 About 區塊第一段更短、更快進入重點

結論：
- 行動版可用
- 下一輪應優先做文案壓縮與段落節奏優化

---

## 7. Visual / Layout

- 目前整體視覺方向穩定，深色背景、卡片式區塊與科技感一致。
- Section hierarchy 基本清楚，但 `About / Bio` 與 `Current Focus` 的視覺差異還可以更明確。
- 入口卡片已成立，但後續若加 `Projects` 或 `Experiments`，版面會更完整。
- 目前仍維持純 HTML + CSS，這個方向應繼續保留，不需要引入框架。

結論：
- 不需要大改視覺風格
- 下一輪可聚焦資訊層級與 section distinction，而不是整體重設計
