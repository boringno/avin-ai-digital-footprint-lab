# 個人網站首頁 Hero 草稿

適用對象：AVIN  
定位主軸：公開建構中的 AI Workflow Explorer  
目的：把 GitHub 上已建立的公開證據、LinkedIn 已完成的對外更新，以及 AVIN 的背景優勢，整理成個人網站首頁第一屏可直接使用的版本

---

## 一、Hero 標題

### 版本 A

```text
公開建構中的 AI Workflow Explorer
```

### 版本 B

```text
用 AI 工具實測與工作流拆解，累積可被看見的數位足跡
```

### 建議採用

如果首頁第一屏要先上一版，建議使用版本 A，最直接，也最能對應目前 GitHub 與 LinkedIn 的公開定位。

我持續學習 AI、研究工具、拆解工作流，並把過程整理成 GitHub 文件、公開紀錄與可長期累積的數位資產。

## 六、首頁呈現原則

這份 Hero 草稿要保留幾個重點：

- 主軸是 AI 工具實測、工作流拆解、GitHub 數位足跡建構
- 語氣是公開建構中，不是假裝成熟完成
- 不把 AVIN 包裝成 AI 大師
- 不回到單純醫美人定位
- 保留內容、行銷、短影音背景作為理解平台與轉譯工具的底層優勢

## 八、補充說明

這份首頁 Hero 草稿的目標，不是把首頁寫得很滿，也不是做成過度行銷的自我包裝。

它要做到的是：讓第一次進來的人，很快理解 AVIN 正在做什麼、為什麼值得看，以及可以從哪三個入口開始理解這套公開系統。
