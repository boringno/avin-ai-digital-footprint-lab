# 你的 AI 工具今天正常嗎？

日期：2026-05-11  

分類：Carousel Final Version / Instagram  

狀態：Final localized carousel version  
Ready for Canva / Figma production

---

## 1. Strategic Position

- 這篇不是單純工具推薦
- 這篇是 AI Workflow Explorer 對 AI 工具基礎設施化的觀察
- 核心觀點：當 AI 變成工作流的一部分，它就不只是工具，而是基礎設施；基礎設施需要被監控

## 2. Platform

- 1080 × 1350

## 3. Final 10-slide Copy + Visual Brief

### Slide 1｜封面

- Slide title: 你的 AI 工具今天正常嗎？
- Main text:

```text
你的 AI 工具
今天正常嗎？
```

- Supporting text:

```text
當 AI 變成工作流的一部分
我們需要的不只是工具清單
還需要狀態雷達
```

- Layout suggestion: 主標大字置中，副標放下半部，形成明確封面層級。
- Visual elements: 深色背景、雷達波紋、狀態燈點、模糊 dashboard 線條。
- Icon / status label: `STATUS CHECK`
- Notes for designer: 第一張要先打中痛點，留白要夠，避免塞太多裝飾。

### Slide 2｜角色轉變前

- Slide title: AI 以前像加分工具
- Main text:

```text
以前 AI 工具
比較像是加分工具
```

- Supporting text:

```text
有用很好
不用也還能工作
```

- Layout suggestion: 左上主句，右下小句，版面保持簡潔。
- Visual elements: 單一工具圖示、低干擾背景。
- Icon / status label: `OPTIONAL TOOL`
- Notes for designer: 這頁是節奏鋪墊，畫面不要太滿。

### Slide 3｜依賴開始形成

- Slide title: 現在越來越多工作流依賴 AI
- Main text:

```text
但現在不太一樣了
```

- Supporting text:

```text
越來越多工作流
會依賴
Claude / OpenAI / Cursor /
GitHub Copilot / Windsurf
```

- Layout suggestion: 上主句、下工具清單，清單可做 stack 排列。
- Visual elements: 工具名稱、節點連線、stack 感。
- Icon / status label: `WORKFLOW STACK`
- Notes for designer: 工具名稱可做視覺節奏，但不要變成工具推薦海報。

### Slide 4｜問題改變了

- Slide title: 問題不再是要不要用 AI
- Main text:

```text
這時候問題就變了
```

- Supporting text:

```text
不是：
今天要不要用 AI？

而是：
我依賴的 AI 工具
現在有沒有正常運作？
```

- Layout suggestion: 用「不是 / 而是」做對比排版。
- Visual elements: 左側舊問題淡化，右側新問題高亮。
- Icon / status label: `NEW QUESTION`
- Notes for designer: 這頁是邏輯轉折點，視覺上要有切換感。

### Slide 5｜常見症狀

- Slide title: 你可能也遇過這些狀況
- Main text:

```text
模型突然變慢
工具一直卡住
回應怪怪的
API 好像不穩
AI coding tool 突然失靈
```

- Supporting text:

```text
問題不是少數情況
而是依賴 AI workflow 後
很容易遇到的日常風險
```

- Layout suggestion: 清單式排版，每條症狀各自成行。
- Visual elements: 延遲圖示、error badge、斷線圖示。
- Icon / status label: `WARNING`
- Notes for designer: 保持手機可讀，單行不宜過長。

### Slide 6｜最麻煩的地方

- Slide title: 最麻煩的不是出錯
- Main text:

```text
最麻煩的
不是出錯
```

- Supporting text:

```text
而是你很難判斷：
是我網路問題？
是平台端異常？
是模型服務出狀況？
還是工具本身正在不穩？
```

- Layout suggestion: 短主標 + 4 行診斷問題。
- Visual elements: 分叉路徑、問號節點、故障排查感。
- Icon / status label: `UNCLEAR SOURCE`
- Notes for designer: 這頁的重點是診斷困難，不是列 bug。

### Slide 7｜AVIN 觀察

- Slide title: 我覺得 Gawk Dev 有趣的地方
- Main text:

```text
我覺得 Gawk Dev
有趣的不是它列了哪些工具
```

- Supporting text:

```text
而是它把 AI coding stack 的即時狀態
集中成一個可以先檢查的入口
```

- Layout suggestion: 上方保留 AVIN 觀察，下方再接 dashboard 感摘要。
- Visual elements: 狀態面板、服務列表、綠黃紅燈號。
- Icon / status label: `STATUS RADAR`
- Notes for designer: 這頁要保留人味，讓觀點不像模板，而像作者自己的判斷。

### Slide 8｜使用場景

- Slide title: 工具失常時先確認什麼
- Main text:

```text
當工具變慢、卡住、
回應怪怪的時候
```

- Supporting text:

```text
你至少可以先確認：
是我這邊的問題？
還是 AI 工具 / 平台端
正在異常？
```

- Layout suggestion: 左右對照或上下二分判斷畫面。
- Visual elements: `Your Side` vs `Platform Side` 對照卡。
- Icon / status label: `CHECK FIRST`
- Notes for designer: 這頁要有 decision aid 的感覺。

### Slide 9｜收觀點

- Slide title: 這提醒我一件事
- Main text:

```text
當 AI 變成工作流的一部分
它就不只是工具
```

- Supporting text:

```text
而是基礎設施
```

- Layout suggestion: 收斂頁，主句大、輔句更大或更重。
- Visual elements: 工具圖示逐漸變成 infrastructure grid。
- Icon / status label: `INFRASTRUCTURE`
- Notes for designer: 這一頁要有觀點落地感，是整組輪播最重要的結論頁。

### Slide 10｜CTA

- Slide title: 基礎設施需要被監控
- Main text:

```text
而基礎設施
需要被監控
```

- Supporting text:

```text
想看這個
AI 工具狀態雷達

留言「AI」
我傳連結給你
```

- Layout suggestion: CTA 置中，留言關鍵字放最大。
- Visual elements: CTA 按鈕感、訊息泡泡、箭頭導向留言區。
- Icon / status label: `CTA`
- Notes for designer: CTA 要單一，不要加第二個行動呼籲。

## 4. Caption Draft

```text
以前 AI 工具對我來說，比較像加分工具。
有它很好，沒有也還能工作。

但現在越來越多 workflow，已經直接依賴 Claude、OpenAI、Cursor、GitHub Copilot、Windsurf 這類工具。

這時候問題就不再是「今天要不要用 AI」，而是「我依賴的 AI 工具現在有沒有正常運作」。

這也是我看到 Gawk Dev 時覺得有意思的地方。
它提醒我：當 AI 變成工作流的一部分，它就不只是工具，而是基礎設施。

而基礎設施，需要被監控。

想看這個 AI 工具狀態雷達，留言「AI」我傳連結給你。
```

## 5. Canva / Figma Design Brief

### Visual direction

- 深色、乾淨、偏 dashboard / infrastructure 感
- 不做花俏科技感
- 重點是清楚、冷靜、可讀

### Color mood

- 深藍 / 深灰底
- 綠、黃、紅作為狀態點綴
- 白字高對比

### Typography direction

- 主標粗、短、集中
- 輔文較小，但行距要寬
- 每頁只保留一個主要閱讀節奏

### Reusable elements

- status label
- dashboard grid
- radar / node / signal dot
- warning badge
- split comparison card

### Slide rhythm

- 1：hook
- 2–4：情境與轉折
- 5–6：痛點與診斷困難
- 7–8：觀察與使用場景
- 9：核心觀點
- 10：CTA

### CTA treatment

- 「留言 AI」做最明顯的視覺焦點
- 不要多 CTA
- 保持可執行、可理解

### Export format

- Instagram carousel
- 1080 × 1350
- PNG export
