# 四大核心 Skills｜AI 數位足跡引擎 3.0

## 1. 這四個 Skills 是什麼

這四個 Skills 不是四個彼此孤立的 prompt。

它們是 AVIN AI 數位足跡 OS 的方法論引擎，負責把「資訊」轉成「理解」，把「理解」轉成「內容」，再把「內容」轉成「長期資產」。

它們的價值不在於替 AVIN 自動發文，而在於：

- 幫助研究拆解更有結構
- 幫助內容轉譯更有平台感
- 幫助公開輸出更可信
- 幫助每一次輸出都能沉澱成可複利資產

## 3. Skill A：AI 研究拆解師

用途：

把一個 AI 主題、工具、趨勢、新聞或靈感拆成可研究、可測試、可輸出的內容素材。

連結：

[ai-research-breakdown-skill.md](./ai-research-breakdown-skill.md)

建議包含：

- 適合輸入：AI 新聞、工具頁、RSS 靈感、GitHub Issue、Notebook 筆記
- 主要輸出：研究框架、測試問題、內容角度、風險點
- 何時使用：看到一個 AI 訊號，但還不知道它有沒有價值時

## 4. Skill B：多平台內容轉譯師

用途：

把一份母內容轉成 LinkedIn、Threads、GitHub、個人網站、履歷網站或短影音腳本。

連結：

[multi-platform-content-skill.md](./multi-platform-content-skill.md)

建議包含：

- 適合輸入：已消化筆記、GitHub 文件、系統總覽、案例復盤
- 主要輸出：多平台版本
- 何時使用：一份內容已經有觀點，需要轉成不同平台語言時

## 5. Skill C：可信度與去 AI 味審查官

用途：

檢查內容是否太像 AI、太空泛、缺乏實測、過度誇大，並補上可信度元素。

連結：

[credibility-review-skill.md](./credibility-review-skill.md)

建議包含：

- 適合輸入：草稿、LinkedIn 文、GitHub README、個人網站文案
- 主要輸出：可信度修正版、風險提醒、待補證據
- 何時使用：內容準備發布前

## 6. Skill D：數位足跡資產官

用途：

判斷一份內容應該沉澱成什麼資產，例如 GitHub 文件、Notion SOP、LinkedIn 貼文、個人網站案例、履歷 bullet、Prompt 庫。

連結：

[digital-asset-review-skill.md](./digital-asset-review-skill.md)

建議包含：

- 適合輸入：學習紀錄、工具實測、案例、內容草稿
- 主要輸出：資產分類、沉澱位置、下一步行動
- 何時使用：決定內容要去哪裡、如何長期複利時

## 7. 四大 Skills 的標準使用流程

```text
AI 靈感雷達 / 學習紀錄
↓
Skill A：AI 研究拆解師
↓
AVIN 人工實測與消化
↓
Skill B：多平台內容轉譯師
↓
Skill C：可信度與去 AI 味審查官
↓
Skill D：數位足跡資產官
↓
GitHub / LinkedIn / 個人網站 / 履歷網站 / 社群
```

## 8. 使用原則

- 不把 Skill 當自動發文機
- 不讓 AI 取代 AVIN 的判斷
- 每次輸出都要保留實測痕跡
- 先沉澱母內容，再轉平台
- 每個輸出都要對應一個長期資產

## 9. 實戰範例

輸入：

`AI 靈感雷達抓到「Gemini API File Search is now multimodal」`

使用方式：

- Skill A 先把這則訊號拆成研究問題，例如：它和既有 file search 差在哪裡、multimodal 的實際價值是什麼、哪些使用情境值得測試
- AVIN 再去讀官方文件或親自實測，確認這不是只有標題好看，而是真的有應用價值
- Skill B 把已消化的內容轉成 LinkedIn、GitHub 文件、Threads 等不同平台版本
- Skill C 在發布前檢查是否有過度誇大、缺少實測證據、語氣太像 AI 自動摘要
- Skill D 最後判斷這份內容最適合先沉澱成 GitHub 文件，再從母內容轉成 LinkedIn 貼文
