# External Skill Evaluation Workflow

Date: 2026-05-11
Type: Workflow / Skill Evaluation
Status: Draft

---

## 1. Purpose

這個流程說明 AVIN 不閉門造車，也不會直接把外部 skill 整包接進來。做法是先實測外部 skill，再決定是否導入、在地化、融合，或直接拒絕。

---

## 2. Core Principle

- 外部 skill 不直接整包採用
- 先測試、比較、本地化
- 保留 AVIN 的 AI Workflow Explorer 觀點

---

## 3. Standard Evaluation Flow

- Test Plan
- Test Material
- External Output
- Comparison Note
- Decision: Adopt / Localize / Merge / Reject
- Localized Final Version
- Production
- Published URL 回寫
- Lifecycle Completed

---

## 4. Decision Types

- Adopt: 直接採用
- Localize: 保留外部優點，但補回 AVIN 語氣與 OS 脈絡
- Merge: 融合進本地 skill
- Reject: 不採用

---

## 5. Evaluation Criteria

- 是否改善工作流
- 是否提升輸出品質
- 是否保留 AVIN 觀點
- 是否太模板化
- 是否能接回 Notion / GitHub lifecycle
- 是否值得成為 OS 的 reusable workflow

---

## 6. Case Example

- Gawk Dev carousel test
- Decision: Localize
- 外部 carousel logic 強在 visual brief / swipe flow / status label
- AVIN 原始版本強在觀點與語氣

---

## 7. What This Avoids

- 不閉門造車
- 不盲目導入
- 不把外部工具當答案
- 不讓工具取代 AVIN 的判斷
