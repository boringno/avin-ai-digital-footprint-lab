# Personal Website Progress Update Spec

## 1. Purpose
The personal website is not a static resume; it is the **Public Showcase Layer** of the AI Digital Footprint OS. Its purpose is to demonstrate AVIN’s role as an AI Workflow Explorer and Strategist by providing tangible "proof of work" for completed milestones.

## 2. Website Role in the OS
The website acts as the central hub for external visibility, connecting various output layers:
- **Notion:** The internal "Nerve Center" for synchronization and tracking.
- **GitHub:** The repository for technical documentation and "Learning in Public" logs.
- **Personal Website:** The "Public Front Door" and Portfolio Proof.
- **LinkedIn:** The professional narrative and career-aligned storytelling.
- **Threads / IG:** Daily progress updates and content format experiments.

## 6. What Not to Claim
To maintain credibility, avoid the following overstatements:
- **Do not claim** full auto-publishing (It is currently manual publish).
- **Do not claim** batch synchronization (It is currently single-row validation).
- **Do not claim** IG / LinkedIn API integration (These are future goals).
- **Do not claim** to have found the "Viral Formula" (We have found an *Experiment Library*, not a formula).

## 7. Update Frequency
Updates to the website should occur whenever a **Lifecycle Milestone** is reached. 
*Example: Moving from Single-row Sync to Batch Sync triggers a "Status" update on the Project Card.*
