Triggers

Route Events Into the Right Lane

Trigger cards replace the long narrative map and make routing decisions easier to scan.

Every trigger keeps route, output, and review expectation visible.

New GitHub repo found

Trigger Event: New GitHub repo or AI tool appears
Route To: Candidate Batch
Output: Project card or watch note
Review Needed: AVIN review needed before any deeper move

Candidate looks promising

Trigger Event: Candidate survives initial architecture fit
Route To: Document Only or Watch
Output: Review note and comparison note
Review Needed: Human review before trial lane entry

Workflow completed

Trigger Event: Workflow, note, or public output reaches completion
Route To: Completed lane and project log
Output: Completion log and indexed evidence
Review Needed: Review quality and decide if reuse potential exists

Need repo-safe execution

Trigger Event: Need file edits, docs updates, or static UI work
Route To: Codex handoff
Output: Prompt, repo-safe task, and completion log
Review Needed: AVIN checks branch, scope, and final diff

Need read-only quality audit

Trigger Event: Need a second pass on clarity, structure, or risk
Route To: Claude Code handoff
Output: Audit notes and recommended fixes
Review Needed: AVIN decides whether to apply suggested changes

Future agent proposal request

Trigger Event: Future discussion about automation or imports
Route To: Hermes proposal layer
Output: Proposal only
Review Needed: Always yes; no write or external action without approval
